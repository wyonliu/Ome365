## 一 · 现状诊断：为什么"企业 AI 知识库"赛道没爆

全行业三年烧了几十亿，还没有一个"人人必装"的企业智能体平台。根本原因不是模型不够强，是这些平台的**三层根基**都缺一块：

| 层 | 缺了什么 | 症状 |
|---|---|---|
| **常识层** | 没有企业实体图 — 只有文档向量索引 | 搜"X"找不到"—"、RAG 乱引、Agent 把一个人当两个人 |
| **事实层** | 没有变动时间轴 — 只有"最新快照" | 上季数据和本季结论混在一起，Agent 没有"这事 3 月之后变过"的意识 |
| **关系层** | 没有人-人-组织-项目的闭环 — 只有元数据字段 | Agent 能查到某人邮箱，答不出"谁会 review 这个 PR" |

**Ome365 的差异化 = 把"常识层"做成一级能力**（EEG）。这是最难被竞品 5 分钟抄走的护城河 — 数据而非算法，越用越准。

## 二 · 爆款企业智能体平台应有的「七件套」

我们已有 ✅ / 缺口 ⚠️ / 路线图有 🟡：

| 能力 | 说明 | 状态 |
|---|---|---|
| ① **Entity Graph (EEG)** | 人/组织/产品/术语 + 别名 + 置信度 + 证据 | ✅ P0 已上线 |
| ② **Temporal Facts** | 事实带时间戳，可查"这话是什么时候说的、谁说的、后来有没有推翻" | 🟡 Memory 雏形 |
| ③ **Multi-Tenant Isolation** | 一个实例服务多个组织，彼此看不到；跨租户脱敏可控 | 🟡 设计中 |
| ④ **Permission-Aware RAG** | 检索按角色切片 — 员工不会搜到董事会纪要 | ⚠️ 未开始 |
| ⑤ **Provenance-First 生成** | 每个答案必须给出证据链 (file:line)，不给不答 | ⚠️ Interviews 有片段引用，RAG 没接 |
| ⑥ **Human-in-the-Loop 审核** | 新实体/新事实自动入 `_pending/`，人一键 Approve | 🟡 EEG Phase 1 |
| ⑦ **Workflow 闭环** | 不仅查询，能触发：开会/发 PR/建任务/发消息 | ✅ Tasks/Reminders 雏形 |

**一句话结论**：有①⑦的已经有（Mem/Notion），有②③的很少，有④⑤⑥的几乎没有。Ome365 要爆，**差的就是把②~⑥一次性打通**。

这五列加起来，是一个做 PKM 的人每天都能体会到的**真实使用门槛**。而不是 PM 讲给投资人的故事。

## 四 · 最佳使用实践（对 B 端客户的说服路径）

### 4.1 落地钩子：从一个痛点切入

不要卖"全能 AI 助手"——企业买的永远是**已经痛到流血的那一刀**。Ome365 的三个首选钩子：

1. **「谁谁说过什么」检索** — 高管开会说过的话找不回来，EEG + 访谈片段引用 3 秒搜出原话
2. **「同事名字 ASR 永远打错」** — 100 人团队里 30 个人的名字会被 Whisper / TicNote / 飞书妙记误读，EEG 一次录入全站通用
3. **「新员工入职 3 天读完组织图」** — EEG + 关系图 + 访谈切片，新人跟一周 10 个关键人物对话的精华一次性灌给他

这三个钩子的共性：**"没有替代品"**。竞品即便号称有，实际体验比 Ome365 差一个数量级，因为他们没有实体层。

### 4.2 采用曲线：从个人到团队到租户

```
Week 1: CTO 一个人用（驾舱+个人助手）— 今天就能跑
Week 2-4: CTO + 3 个 BU 负责人 共享 EEG — 给权限、加术语
Month 2-3: 全员试点：每人个人首页 + 本 BU 视图
Month 4+: 全租户正式部署 + 权限分层
```

**关键指标**：
- **Day-1 能交付 1 件"真的有用"的事**（否则不再第二次打开）
- **Week-1 EEG 命中率 > 80%**（说一个人名，系统能召回正确档案）
- **Month-1 发生一次"没有 Ome365 就完蛋"的瞬间**（找回某个决策/某次会议原话）

### 4.3 运营节奏：AI 一二线双轨

- **一线（每个员工）**：个人首页 + 检索 + 闲聊问事实
- **二线（CTO/管理层）**：驾舱 + 跨 BU 总览 + 战略看板
- **值班（EEG 审核员 1-2 人/1000 员工）**：处理 `_pending/` 队列，保持实体层干净

## 五 · 爆款触发条件 · 逻辑推理

一个 B 端产品要爆，必须**同时**满足：

1. **替代了一个活的工作流**（不是多一个工具）
2. **每个用户的每一次使用都在为下一次使用增加价值**（数据飞轮）
3. **撤掉它，团队会抱怨**（痛感 > 装机量）

Ome365 的打法：
- **替代了**：高管/CTO 的"访谈-笔记-决策-回溯"工作流（今天他们用飞书妙记 + 石墨文档 + 微信 + 脑子）
- **飞轮**：每场访谈自动扩充 EEG，EEG 越精准下一场转录越准，转录越准决策回溯越快
- **痛感**：EEG 一停，所有人的名字又开始打错，所有术语又开始跟 LLM 解释

## 七 · 一句话回答"你们到底比 Glean 强在哪"

> **Glean 让你找到文档里的一句话；Ome365 知道这句话是谁说的、什么时候说的、后来有没有被推翻、现在由谁负责跟进。**

这一句话差 4 个实体/时间/修订/责任人维度——就是企业真正的"知识"和"文档"的差距。

Glean 是"企业 Google"。Ome365 是"企业大脑"。
